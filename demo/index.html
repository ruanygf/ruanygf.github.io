arch 使用 hugo +github 搭建博客

27 October 2025·307 words·2 mins

参考链接：https://krislinzhao.github.io/docs/create-a-wesite-using-github-pages-and-hugo/

整体思路
#

github 上建立 2 个仓库，一个存放源码，一个存放构建内容，使用 github 自带的 action 功能，将代码提交到源码仓库能自动渲染同步到渲染仓库

远程仓库创建
#

源码仓库
#

名字自取，推荐和之后建立的项目名一致，==设置私有==，如我的博客名称是“yaminook”，因此取这个名字，后续本地建立的项目名称也是这个名字

目标仓库
#

名称必须为“用户名.github.io”，==设置公有==

创建部署密钥（实现仓库间免密推送）
#

需要创建 SSH 密钥，让源文件仓库的 Actions 有权限推送文件到 ruanygf.github.io 仓库。

在本地终端生成 SSH 密钥对（无需设置密码，一路回车）：

ssh-keygen -t ed25519 -C "hugo-deploy" -f ~/.ssh/hugo_deploy_key

执行后会生成两个文件：

  • 私钥：~/.ssh/hugo_deploy_key（保密，仅配置在源文件仓库）
  • 公钥：~/.ssh/hugo_deploy_key.pub（配置在 GitHub Pages 仓库）

配置公钥到 ruanygf.github.io 仓库（目标仓库）：

  • 打开 GitHub 上的 ruanygf/ruanygf.github.io 仓库 → 进入「Settings」→ 「Deploy keys」→ 「Add deploy key」。
  • Title 填「hugo-deploy-key」，Key 粘贴 hugo_deploy_key.pub 文件的全部内容，勾选「Allow write access」（允许写入），点击「Add key」。 配置私钥到源文件仓库：
  • 打开 GitHub 上的 ruanygf/yaminook 仓库 → 进入「Settings」→ 「Secrets and variables」→ 「Actions」→ 「New repository secret」。
  • Name 填「DEPLOY_KEY」（必须与后文中 deploy.yml 中的 ${{secrets.DEPLOY_KEY}} 一致），Secret 粘贴 hugo_deploy_key 文件的全部内容，点击「Add secret」。

安装 hugo
#

sudo pacman -S hugo # 我此时安装的是0.15.2版本

创建网站
#

# 创建项目
hugo new site YamiNook # YamiNook为项目名

# 安装主题，以Blowfish为例，在 https://github.com/nunocoracao/blowfish 查看安装流程
git init
git submodule add -b main https://github.com/nunocoracao/blowfish.git themes/blowfish
rm hugo.toml # # 删除 Hugo 自动生成的默认配置文件
mkdir -p config/_default # 创建存放主题配置的目录（有就不创建）
cp themes/blowfish/config/_default/*.toml config/_default/ # 复制主题中的所有 .toml 配置文件到项目配置目录

# 配置主题
# 打开 config/_default/hugo.toml：设置网站标题（title）、语言（defaultContentLanguage）、作者信息等。其他配置文件（如 menu.toml、params.toml）：可根据需求调整导航菜单、主题参数（如颜色、社交链接等）。
# 最基本要修改 config/_default/hugo.toml中两个位置：
baseURL = "https://example.com/" #把https://example.com/改成自己的域名，如： https://ruanygf.github.io/ (注意以/结尾)
theme = "blowfish" # 取消注释这一行

# 创建文章
hugo new xxx.md # 编辑时如果想在网站中显示设置 draft: true

# 预览内容
hugo server # 浏览器中使用 localhost:1313查看

本地和远程建立连接
#

# 本地在 yaminook 文件夹下执行
git remote add origin git@github.com:ruanygf/yaminook.git
git branch -m master main # 将默认分支修改为main与远程保持一致

# 添加配置文件
mkdir -p .github/workflows
touch .github/workflows/deploy.yml  # 新建部署脚本文件

将以下内容粘贴到 deploy.yml 中（替换关键信息）

name: 自动部署 Hugo 网站到 GitHub Pages

# 触发条件：当推送到 main 分支时执行
on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest 
    steps:
      - name: 检出源文件
        uses: actions/checkout@v4
        with:
          submodules: true  
          fetch-depth: 0

      - name: 安装 Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: '0.151.2'  #（版本与本地一致，可通过 hugo version 查看）
          extended: false  # Blowfish 无需扩展版，保持 false

      - name: 构建网站
        run: hugo --minify  

      - name: 部署到 ruanygf.github.io
        uses: peaceiris/actions-gh-pages@v4
        with:
          external_repository: ruanygf/ruanygf.github.io # 根据自己的github博客名称修改
          deploy_key: ${{ secrets.DEPLOY_KEY }} # 根据前文设置的Action名称修改
          publish_dir: ./public
          publish_branch: main

推送代码到源码仓库

git add .
git commit -m "xxxx"
git push origin main

查看部署状态：

  • 进入源文件仓库 → 「Actions」→ 查看最新的 workflow 运行记录，若显示绿色对勾则部署成功。
  • 等待一小会会发现ruanygf.github.io中同步了public目录下的内容，访问 https://ruanygf.github.io，即可看到自动构建的网站（包含测试文章）。
